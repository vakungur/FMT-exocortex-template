# Локальный LLM-стек IWE (РП404)

Локальная языковая модель на Mac (Apple Silicon, движок MLX) под две работы: **приватность** (задачи с персональными данными, что нельзя слать в облако) и **запасной режим** (без сети или когда кончились токены облака). Решение и обоснование — в `ADR-001-local-llm-stack.md`.

## Установка (одна команда)

```bash
bash install-local-llm.sh          # рекомендованная модель под твоё железо
bash install-local-llm.sh --max    # самая тяжёлая модель, влезающая в память
```

Установщик идемпотентен: детект железа → выбор модели из каталога под объём памяти → окружение + mlx-lm → скачивание модели → проверка. Повторный запуск ничего не ломает.

## Работа со стеком

```bash
bash iwe-local-llm.sh start          # поднять сервер (OpenAI-совместимый, localhost:8080)
bash iwe-local-llm.sh status         # активная модель + состояние сервера
bash iwe-local-llm.sh test           # проверка совместимого интерфейса
bash iwe-local-llm.sh models         # все модели по четырём состояниям + факт скачивания
bash iwe-local-llm.sh pull <model>   # скачать модель для тестирования (status -> testing)
bash iwe-local-llm.sh use <model>    # поставить модель в работу (скачать + активировать + перезапуск)
bash iwe-local-llm.sh archive <model># вывести модель в архив
bash iwe-local-llm.sh stop           # остановить сервер
```

Инструменты IWE подключаются к `http://127.0.0.1:8080/v1/chat/completions` тем же клиентом, что и к облаку.

## Жизненный цикл модели (четыре состояния)

Каждая модель в каталоге имеет статус, модель проходит путь:

| Состояние | Что значит |
|-----------|-----------|
| **в анализе** (`analysis`) | найдена монитором, на рассмотрении; ещё не скачана |
| **для тестирования** (`testing`) | скачана, сравнивается с активной |
| **активная** (`active`) | стоит в работе, сервер поднимает именно её; ровно одна |
| **архивная** (`archived`) | выведена из обращения (устарела/проиграла сравнение) |

Источник истины — `model-catalog.yaml` (правится с сохранением комментариев через `model-lifecycle.py`). Команда `models` показывает все четыре корзины и реальный факт скачивания (по кэшу). При `use <model>` прежняя активная автоматически уходит в `testing` (остаётся скачанной).

## Поиск новых моделей (раз в две недели)

```bash
python3 model-monitor.py            # дайджест трендовых моделей, которых нет в каталоге
```

Монитор сверяет каталог с трендами Apple-Silicon-моделей (HuggingFace). Решение о добавлении кандидата в каталог — за пилотом (ручная сверка качества по LMArena). Найдёшь лучше текущей — добавь в `model-catalog.yaml` и `use <model>`.

## Безопасность (мини-проверка B7.3)

- Сервер слушает только `127.0.0.1` — наружу не торчит.
- Движок MLX работает оффлайн, телеметрию не шлёт.
- В журнал сервера пишутся счётчики токенов и адрес запроса, не содержимое промптов.
- Запасной движок Ollama имеет телеметрию — установщик обязан её блокировать (на пути fallback).

## Требования

macOS на Apple Silicon, `uv` (или `python3.12`), ~5-10 ГБ на диске под модель.
