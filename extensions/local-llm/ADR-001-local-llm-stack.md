# ADR-001 — Стек локальных LLM на Mac (РП404)

> Decision record (лёгкий, не ArchGate). Источник: peer-сессия 2026-06-10-26-local-llm-stack-mac (Claude + Kimi).
> Дата: 2026-06-10. Статус: accepted.

## Контекст

Весь IWE работает на облаке (Claude, Kimi). Локальная LLM на Mac пилота — новая роль в системе. До выбора модели надо назвать её JOB.

## JOB (для чего локальная модель)

**Primary (co-primary, совместимы на уровне deployment — оба локальны, без облака):**
1. **Приватность** — задачи с PII (чистка секретов, персональные данные), которые нельзя отправлять в облако. Облако это не закрывает без enterprise-контракта.
2. **Fallback** — работа без интернета или когда исчерпаны токены облака. Деградированный режим: подмена облака на простых задачах.

**Secondary (бонус, если железо позволяет):** дешёвый bulk. НЕ драйвер выбора — закрывается облачными тирами (Haiku, Flash).

**Вынесено из РП404 (spin-off):** бесплатная модель на сервере платформы для массовых задач (ДЗ-чекер) — другой target (сервер, не Mac), другой JOB (bulk для пользователей платформы), срабатывает IntegrationGate + вопрос стоимости хостинга. → quick-feasibility spike (1-2h) в рамках WP-337 → потом отдельный РП через WP Gate.

## Железо (детект 2026-06-10)

| Параметр | Значение |
|----------|----------|
| Chip | Apple M4 (Mac16,13) |
| Unified memory | 32 GB |
| macOS | 26.3.1 |
| CPU cores | 10 |
| Disk free | 93 GB |
| Python (системный) | 3.14.3 (homebrew) |
| Рантаймы | отсутствуют (нет MLX, Ollama, LM Studio) |

> Окружение стека (venv) собирается на **Python 3.12** (`IWE_LLM_PYVER`, дефолт установщика) — под него гарантированы колёса MLX; системный 3.14.3 свежий и для MLX ненадёжен. Это разные вещи: 3.14.3 — что нашли в системе, 3.12 — на чём собран venv.

## Решения

### 1. Рантайм: MLX primary, Ollama fallback
- **MLX** (native Apple Silicon) — быстрейший на M-series, **оффлайн по природе, не шлёт телеметрию** (ключевое для JOB приватность — B7.3 «из коробки»).
- Ollama — fallback (проще, больше готовых моделей), но имеет телеметрию → требует явной блокировки. На fallback-пути установщик обязан жёстко выставить env, отключающий телеметрию.

### 2. Модель: одна general ~7-8B, on-demand без keep-alive
- **Trade-off (явно):** жертвуем оптимальностью лёгкой PII-модели и мгновенностью fallback ради простоты «одна модель, один установ». Fallback — деградированный режим, разовая загрузка 15-30 сек приемлема.
- **Ветка по RAM (детект решает, не угадывание):**
  - RAM ≤ 16GB → одна general ~7-8B, on-demand. Без второй модели.
  - **RAM ≥ 32GB (наш случай) → одна general ~7-8B как MVP; опционально вторая лёгкая (3-4B) для мгновенной приватности — НЕ обязаловка, по желанию пилота позже.**
- **MVP-модель:** `mlx-community/Qwen2.5-7B-Instruct-4bit` (~4.3GB, влезает с запасом в 32GB и 93GB диска). Сильная general-модель, хорошо поддержана mlx-community.

### 3. Интеграция: OpenAI-compatible API shim (обязательно)
- Без совместимого endpoint локаль = standalone-игрушка. Существующие инструменты IWE должны переключаться на локаль тем же клиентом, что и на облако.
- Реализация: `mlx_lm.server` (OpenAI-compatible `/v1/chat/completions`) на localhost. Это превращает локаль в реальный тир Model Tiering (WP-337).

### 5. Жизненный цикл модели (четыре состояния)

Каждая модель в каталоге имеет статус: `analysis` (на рассмотрении, найдена монитором) → `testing` (скачана для сравнения) → `active` (в работе, ровно одна) → `archived` (выведена). Источник истины — `model-catalog.yaml` (правится с сохранением комментариев через `model-lifecycle.py`). Факт скачивания проверяется по кэшу HuggingFace (реальность, не флаг). Инвариант «ровно одна active» держится на путях `use`/`set active`/`add` (прежняя active демотится в `testing`).

### 4. Мини-чек безопасности B7.3 (JOB приватность затрагивает PII)
- [ ] Рантайм не шлёт телеметрию наружу (MLX — нет; Ollama fallback — env блокировка).
- [ ] `mlx_lm.server` / обёртка не пишут prompt/PII на диск (явный аудит логов, не «по умолчанию наверное не пишет»).
- [ ] Сервер биндится только на localhost (127.0.0.1), не на 0.0.0.0.

## Отвергнуто

- **Полный ArchGate** — непропорционален персональному инструменту на одной машине (ЭМОГССБ прошёл бы тривиально). Лёгкий ADR достаточен.
- **Фиксированная «золотая» модель в FMT** — неработоспособна при разном железе пилотов («Костюм ≠ Оснащение», DP.D.131). Раздаём процесс выбора + установщик.
- **llama-2 / claude.ai как кандидаты** (исходный критерий РП) — llama-2 устарел на 3 поколения; claude.ai категориально не локальна.
- **Еженедельный мониторинг** — избыточен; biweekly соответствует циклу MLX-quantized релизов (2-3 недели).
