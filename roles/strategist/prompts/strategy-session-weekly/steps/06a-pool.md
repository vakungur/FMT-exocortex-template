# Step 06a: План — candidate pool

## ВДВ

| | |
|---|---|
| **Вход** | Черновик WeekPlan, активные R{N} месяца, carry-over, предложения из inbox, новые РП из калибра (05b) |
| **Действие** | Сформировать пул кандидатов, проставить приоритеты |
| **Выход** | Candidate pool с приоритетами (🔴/🟡/🟢/⚪) |

## Инструкция для Claude

> Метод: `PD.METHOD.017` Work Product Selection.

1. **Candidate pool** — пройди предложенные РП из черновика session-prep:
   - (a) активные R{N} месяца
   - (b) carry-over с приоритизацией
   - (c) предложения из inbox triage (шаг 02)
   - (d) новые РП из калибра (шаг 05b, если созданы)

2. **Режим работы (триада) — две независимые оси, не путать.**

   **Ось 1 — режим ТВС (текущее / важное / срочное):** для каждого РП определи режим по конвейерной модели (см. различение [[Текущее ≠ Важное ≠ Срочное (ТВС)]]):
   - 🔥 **Срочное** — угроза остановки конвейера (прод упал, биллинг сломан, блокирует других агентов/пользователей, потеря данных, security-critical). **Только это.** Срочное в норме НЕ планируется в неделю — оно прерывает и чинится немедленно; если попало в пул, значит конвейер уже стоит.
   - 🟡 **Важное (развитие)** — поднимает норму выше прежней (новая способность, модернизация, стратегический R{N}). Это сердцевина стратегической недели: важное системно вытесняется текучкой и ложно-срочным, поэтому защищается осознанно.
   - 🟢 **Текущее (текучка)** — штатная операционная работа, держит конвейер в плановой норме (поддержка, рутина, операционные НЭП).

   **Ось 2 — дедлайн (ортогонально режиму):** есть ли согласованный момент передачи дальше? Дедлайн может стоять на ЛЮБОМ из трёх режимов и сам по себе НЕ делает РП срочным (см. [[Дедлайн ≠ Срочность]]). Помечай дедлайн отдельной меткой `⏰ до DD.MM`, не повышай им режим.

   **Критическая проверка (частая ошибка):** «приближается дедлайн» ≠ «срочное». Сильная неудовлетворённость / высокая мотивация ≠ «срочное» — это важное. Срочное = только физическая угроза остановки конвейера.

3. **Приоритет = функция режима + дедлайн + ТОС.** Порядок взятия в неделю: реальное 🔥срочное (если ещё не закрыто) → 🟡важное под ТОС недели и активные R{N} → 🟡важное с близким ⏰дедлайном → 🟢текучка (минимально необходимая для удержания нормы). Цель недели — защитить квоту 🟡важного, а не дать текучке и ⏰дедлайнам вытеснить развитие.

4. **Вопрос пилоту:**
   > «Продолжить к бюджетированию / скорректировать / пропустить?»
